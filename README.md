# MoFox-Core 一键管理程序 (Linux)

## 📖 项目概述

MoFox-Core 一键管理程序是一个专为 Linux 环境设计的 Python 管理工具，旨在简化 MoFox-Core 机器人系统的部署、运行和维护。该程序提供了友好的命令行界面，支持一键启动、更新和管理核心服务。

## ✨ 主要功能

### 🚀 服务管理
- **多服务启动**: 支持启动 MoFox Core 主程序、Napcat 服务和 Matcha 程序
- **状态监控**: 实时查看所有服务的运行状态和进程信息
- **安全停止**: 一键安全停止所有正在运行的服务

### 🔄 版本控制与更新
- **GitHub集成**: 支持仓库的自动更新
- **智能更新**: 自动检测本地与远程仓库的差异，显示落后的 commit 信息
- **依赖管理**: 更新仓库后自动安装/更新相关依赖包

### 🛠️ 系统维护
- **依赖安装**: 智能安装和更新所有服务的 Python 依赖包
- **权限修复**: 自动诊断和修复 pip 权限问题
- **系统信息**: 显示详细的系统和环境信息
- **仓库状态**: 检查仓库的 commit 状态和更新情况

## 🎯 支持的服务

| 服务名称 | 类型 | 描述 | 仓库地址 |
|---------|------|------|----------|
| MoFox Core 主程序 | Python | AI聊天机器人核心程序 | MoFox-Studio/MoFox-Core |
| Napcat 服务 | Shell | QQ协议服务 (Linux) | 本地服务 |
| Matcha 程序 | Executable | Matcha客户端程序 (Linux) | 本地程序 |

## 📋 系统要求

- **操作系统**: Linux (Ubuntu/Debian/CentOS 等)
- **Python**: 3.8+ (建议使用虚拟环境)
- **Git**: 用于仓库管理和更新
- **终端**: GNOME Terminal, Konsole, XFCE4 Terminal 或 XTerm (用于独立窗口运行服务)

## 🚀 快速开始

### 1. 首次安装与初始化
如果您是第一次使用，请直接运行初始化脚本，它会自动配置环境并安装依赖：

`ash
chmod +x linux首次启动点我.sh
./linux首次启动点我.sh
`

### 2. 日常启动
使用启动脚本快速运行管理程序：

`ash
chmod +x linux启动点我.sh
./linux启动点我.sh
`

或者直接使用 Python 运行：

`ash
source .venv/bin/activate
python3 mofox-core.py
`

## 🔧 高级功能

### 自动更新机制
程序支持 GitHub 仓库的安全更新：
- 自动检测本地与远程差异
- 显示详细的 commit 更新信息
- 更新后自动安装新依赖

### 进程管理
- 服务尝试在独立的终端窗口中运行，方便查看日志
- 如果没有图形界面，将自动在后台运行
- 支持实时状态监控

## ⚠️ 注意事项

1. **权限要求**: 脚本需要执行权限 (chmod +x)
2. **网络依赖**: 仓库更新需要稳定的网络连接
3. **虚拟环境**: 程序默认使用 .venv 目录下的虚拟环境
